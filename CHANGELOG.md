# 版本变更日志 (CHANGELOG)

> 本文档记录 TikTok Ad Video Skill 从初始版本至今的所有重要变更。
> 版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范：主版本号.次版本号.修订号。


## [v2.15.1] - 2026-05-24

### 🚀 新增
- **核心层量化阈值**：在 `evaluation-rubric.md` 和 `SKILL.md` 的三层诊断中，为核心层增加量化标准。垂直领域信号缺失扣 5 分，原生感策略未启用扣 3 分，累计扣分 ≥5 分即不通过。解决之前核心层“警告+微调”缺乏明确阈值的问题。
- **SKILL-lite 重新定位**：明确 `SKILL-lite.md` 为“ChatGPT / Claude / 其他非 DeepSeek AI 平台的独立完整版”，与 `SKILL.md`（DeepSeek 路由器版）形成清晰分工。两者核心工作流、钩子库、评分标准和输出规范完全相同，区别仅在于知识库调用方式。
- **README 加载指南分区**：加载指南从单一模式改为按 AI 平台分区（DeepSeek / 其他 AI），明确各平台推荐使用哪个文件及加载方式。

### 🔄 变更
- **三层诊断表格升级**：在 `SKILL.md` 和 `SKILL-lite.md` 中，三层诊断表格新增“量化标准”列，替代原来的纯描述性检查项。
- **失败模式细化**：在 `evaluation-rubric.md` 失败模式速查表中，将核心层不通过拆分为“垂直信号缺失”和“原生感未启用”两条独立修复方案。
- **核心层量化细则文档化**：在 `evaluation-rubric.md` 中新增 §1.1 核心层量化细则和 §1.2 与五维评分表的对应关系。
- **references/ 版本号标注统一**：`references/` 目录下所有参考文件在本次迭代中版本号统一标注为 v2.15（内容无实质性变更），消除之前版本号不一致的混乱。

## [v2.15] - 2026-05-24

### 🚀 新增
- **DeepSeek System Prompt 深度适配**：`SKILL.md` 从 800+ 行“百科全书”全面重构为约 150 行“路由器”，Token 消耗降低约 85%。新增工作流索引表（每阶段标注跳过条件和参考文件），新增结构化快速诊断矩阵替代文本决策树。
- **Seedance 2.0 官方标准全面对齐**：`cinematic-vocabulary.md` 新增“官方6步公式”专题（主体→动作→环境→镜头→风格→约束），新增“最佳长度建议”（30-60词推荐区间），新增“正/负向词对照表”明确 Seedance 2.0 不支持否定式负向词的核心规则。
- **原子化分段指令输出规范**：`SKILL.md` 新增 §6 输出规范，强制要求直给型提示词输出为独立镜头段落（每段 30-60 词，绝对不超过 100 词），叙事型每段 ≤100 词。规范中明确单镜头单一运镜原则。
- **微短剧系列化脚本模板库**：新增 `references/short-drama-series-guide.md`，包含角色设定、世界观构建、单集结构设计（1分钟左右）、长效钩子设计、品类适配指南及转化点植入时机策略。
- **预评估机制简化**：`SKILL.md` 将阶段 3.5 从“五维精细打分”升级为“三层快速诊断”（致命层/核心层/卫生层），大幅减少评估 Token 消耗。`evaluation-rubric.md` 同步新增 AI 自动执行与人工复盘的职责分离说明。
- **最佳发布时间窗口指南**：`platform-specs.md` 新增 §九，为 8 大平台逐一提供基于 2026 年用户活跃度数据的最佳发布时间、次优时段和避免时段建议。
- **短视频SEO深度策略**：`platform-specs.md` 新增 §十，包含各平台 SEO 权重排序、关键词研究四步流程及 Lemon8 种草型标题模板。
- **AIGC 标签流量影响对照实验设计**：`platform-specs.md` 新增 §十二，提供完整的实验步骤、结果记录模板和决策建议。
- **否定式负向词全面替换**：`cinematic-vocabulary.md` 新增 §三「正/负向词对照表」，系统性地将所有否定式负向词替换为正向锚点词。

### 🔄 变更
- **SKILL.md 架构重构**：工作流从“详细描述”全面改为“结构化索引表”，每阶段明确标注跳过条件和参考文件。
- **提示词约束标准升级**：从“建议”升级为“硬性规定”。明确禁止否定式负向词，强制使用正向锚点词。
- **评估体系职责分离**：AI 自动执行时优先使用三层快速诊断，人工复盘时使用五维评分表。
- **快速诊断矩阵重构**：从文本格式决策树改为结构化列表。
- **成本控制章节更新**：Token 消耗预估数据同步更新。
- **自检报告升级**：新增“否定式负向词检查”“原子化分段词数统计”等检查项。
