# 版本变更日志 (CHANGELOG)

> 本文档记录 TikTok Ad Video Skill 从初始版本至今的所有重要变更。
> 版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/) 规范：主版本号.次版本号.修订号。


## [v2.12] - 2026-05-23

### 🚀 新增
- **评分表分档阈值**：在 `evaluation-rubric.md` 中，将 Prompt 预评估结果从单纯的通过/不通过细化为四档（优秀/合格/需小修/需大修），并明确各档位对应的执行动作。
- **非线性惩罚规则**：当钩子强度(H)或声音策略(A)得分低于 50 分时，无论总分多少，评级自动降一档，确保核心要素不被平均。
- **评分表动态校准机制（后台）**：在 `SKILL.md` 阶段 6 中新增数据记录与回归校准逻辑，当样本量 ≥50 条时自动计算优化权重，实现评分体系的自我进化。
- **预评估执行模板优化**：输出格式中增加评级标识（优秀/合格/需小修/需大修），使用户对下一步动作一目了然。
- **失败模式细化**：在 `evaluation-rubric.md` 失败模式速查表中，将 Prompt 预评估不达标拆分为“需大修”和“需小修”两类，分别给出迭代路径。

### 🔄 变更
- `evaluation-rubric.md` Prompt 预评估章节增加 §1.3 分档阈值表与非线性惩罚说明，更新执行模板示例。
- `SKILL.md` 阶段 3.5 增加四档评级输出格式，阶段 6 增加第 4 条「评分表权重动态校准」后台记录逻辑。
- 工作流图中的判断节点从“评分 ≥80？”更新为“评级：优秀/合格？”，更清晰反映分档逻辑。
- 版本号统一至 v2.12（SKILL.md、evaluation-rubric.md、CHANGELOG.md）。

### 📚 文档更新
- `references/evaluation-rubric.md` → v2.12
- `SKILL.md` → v2.12
- `CHANGELOG.md` → v2.12

## [v2.11] - 2026-05-23

### 🚀 新增
- **阶段 3.5：Prompt 文本预评估（积分风控闸门）**：在消耗即梦 AI 积分前，强制对 Prompt 文本进行五维度打分（满分 100），≥80 分才允许提交生成视频。实现“先验证、后投入”的零成本迭代闭环。
- **Prompt 文本质量评分表**：在 `evaluation-rubric.md` 中新增独立章节，覆盖前 3 秒钩子强度(30分)、声音策略明确度(25分)、垂直领域信号(20分)、原生感设计(15分)、去 AI 味指令完整性(10分)。

### 🔄 变更
- `SKILL.md` 工作流图新增阶段 3.5 节点与迭代循环路径。
- 核心铁律第 10 条更新为“先验证、后投入”。
- 成本控制章节强化积分风控说明。
- 自检报告增加 Prompt 预评估分数检查项。
