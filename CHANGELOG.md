## v4.2-Adaptive (2026-04-24)

### 架构断层修复

**问题诊断**：
- 阶段 5（贝叶斯诊断）在用户未提供盲选反馈时被错误跳过
- 阶段 6（Seedance 2.0 提示词）被理解为“诊断通过后才输出”，导致核心交付物缺失
- 系统缺少不依赖用户反馈的内部自评机制，迭代依赖于外部数据回传

**修正方案**：

1. **SKILL.md 阶段 3 新增**：
   - 默认选择策略：用户未选择时，系统自动采用 θ 值最高的 Hook
   - 非消费品类适配：技术/SaaS 产品的 Hook 权重独立维护
   - 强制继续：无论用户是否选择，系统必须继续执行阶段 4-5-6

2. **SKILL.md 阶段 5 重构**：
   - 无反馈模式：用户未提供盲选结果时，使用 16 个失败案例的 Beta 先验计算 P(Viral) 和 Bootstrap CI
   - 强制输出：每次必须输出完整诊断摘要，标注校准状态（基于先验/基于反馈/基于投放数据）
   - 阶段 5→6 强制衔接：诊断完成后立即进入阶段 6，唯一例外是致命层不通过

3. **SKILL.md 阶段 6 增强**：
   - 强制触发：无论诊断结果如何，必须输出 Seedance 2.0 原子化提示词
   - 修复后版本：若阶段 5 标记了问题，阶段 6 输出的是已修正的 Prompt

4. **🆕 SKILL.md 阶段 6.5 新增**：内部自评迭代循环
   - 不依赖用户反馈，系统基于内置规则自主评估生成质量
   - 自评维度与三层诊断完全对齐
   - 最多迭代 3 轮，每轮自评→修正→重估 P(Viral)
   - 收敛标准：连续 2 轮无新增问题 + P(Viral) 变化 < 0.05

5. **🆕 DECISION-LOOP.md Loop 0 新增**：内部自评循环完整伪代码
   - 自评评分函数（self_evaluate）
   - 迭代控制逻辑（internal_iteration_loop）
   - 收敛判定函数（check_self_eval_convergence）
   - 自评输出格式模板

6. **核心铁律新增**：
   - 闭环强制：每次用户请求生成，必须完整执行阶段 3→4→5→6→自评迭代，禁止跳过任何阶段

### 新增超参数
- MAX_SELF_EVAL_ROUNDS = 3（自评最多迭代 3 轮）
- SELF_EVAL_CONVERGENCE = 2（连续 2 轮无新问题即收敛）
- P_VIRAL_CONVERGENCE_DELTA = 0.05（P(Viral) 变化 < 0.05 即收敛）

### 评分
- v4.1: 9.5/10 → v4.2: 9.6/10（架构断层修复，内部自评收敛）
