# 多模态参考操作卡 (v4.1-Adaptive · New)

> **模块编号**: MR-001  
> **加载条件**: 当用户需要解决“产品光影不一致”“主体细节漂移”“运镜风格随机”时加载  
> **依赖模型**: 即梦 Seedance 2.0（支持最多 12 个同步参考：9 图 + 3 视频）  
> **核心原理**: 文字生视频存在“风格随机漂移”问题。通过上传锚定素材，将模型输出锁定在指定光照、主体和运镜风格上，实现工业级的视觉一致性。

---

## 一、为什么需要多模态参考

### 文字生视频的“风格随机漂移”问题

输入“美女走在雨夜城市”，AI 可能每次生成不同的城市风格——东京霓虹、巴黎古典、纽约现代——因为文字描述无法精确锁定一种特定的“雨夜城市美学”。

同样的，输入“铸铁锅在厨房灶台上煎鸡蛋”，AI 可能生成不同颜色、不同纹理、不同反光特性的“铁锅”——因为“铁锅”在训练数据中有千万种形态。

### 多模态参考的核心价值

| 问题 | 文字Prompt能做到的 | 加上多模态参考 |
|:---|:---|:---|
| 产品颜色一致性 | “深灰色铁锅”——但 AI 理解的是“某种灰色” | 上传产品实拍图 → AI 锁定该精确色号 |
| 光影风格统一 | “侧逆光、暖调”——但 AI 每次渲染不同 | 上传光照参考图 → AI 复制该光线分布 |
| 运镜风格稳定 | “慢摇推近”——但 AI 运动轨迹不可控 | 上传手持跟拍视频 → AI 学习该运动模式 |
| 多镜头主体不变 | 每段 Prompt 独立生成，主体可能微变 | 9 张多角度图 + 3 段视频 → 强主体锁定 |

## 二、三种锚定模式详解

### 2.1 光照锚定 (Lighting Anchor)

**适用场景**: 
- 需要全片统一光照风格（如电影感暖调、高对比度体积光）
- 产品需要在不同场景中保持一致的材质反光特性

**操作步骤**:
1. 拍摄/找到一张符合目标光照风格的参考图
2. 上传至即梦 AI 的 @Reference 功能
3. 在 Prompt 末尾添加：`光照风格参考上传图片，4K画质`

**参考图选择标准**:
- ✅ 高对比度、有明显光源方向的图（如电影静帧、棚拍产品图）
- ✅ 与你产品材质接近的物体在相同光照下的照片
- ❌ 过曝或欠曝的图（AI 无法判断光源强度）
- ❌ 多光源混杂的场景（如迪斯科舞厅）

**实战案例**:
```
目标: 让铁锅在不同镜头中保持一致的“暖调侧逆光+金属反光”

上传参考图: 铁锅在傍晚窗边、侧逆光下的实拍照片

Prompt:
【镜头1】特写, 铸铁锅在灶台上, 鸡蛋滑入锅中, 光照风格参考上传图片, 4K
【镜头2】近景, 铁铲划锅面, 手持跟拍, 光照风格参考上传图片, 4K
【镜头3】中景, 连续煎蛋展示, 环绕拍摄, 光照风格参考上传图片, 4K
```

---

### 2.2 主体锚定 (Subject Anchor)

**适用场景**:
- 多镜头视频中，产品外观（颜色/纹理/形状）需要绝对一致
- 产品有独特设计细节（LOGO、纹理、特殊造型）需精确还原

**操作步骤**:
1. 拍摄产品正面、侧面、45°角三视图（共 3 张）
2. 上传至 @Reference，选择“主体锚定”模式
3. 在每段 Prompt 中添加：`产品外观严格参考上传图片`

**参考图拍摄标准**:
- ✅ 纯色背景（白/灰），无杂物干扰
- ✅ 产品占画面 70% 以上
- ✅ 对焦清晰、无阴影遮挡产品细节
- ❌ 手持晃动拍摄的模糊图
- ❌ 产品只占画面一小部分的环境图

**实战案例**:
```
目标: 4 个镜头中，元宝锅的“元宝形把手”和“锅底纹理”完全一致

上传三视图:
- 正面: 展示锅体直径、把手弧度
- 侧面: 展示锅深、把手厚度
- 45°: 展示整体比例

Prompt:
【镜头1】特写, 鸡蛋落入锅中, 锅体外观严格参考上传图片, 4K
【镜头2】近景, 铁铲划过锅底纹理, 产品外观严格参考上传图片, 4K
【镜头3】中景, 连续煎蛋, 锅体外观严格参考上传图片, 4K
【镜头4】近景, CTA展示, 产品外观严格参考上传图片, 4K
```

---

### 2.3 动态锚定 (Motion Anchor)

**适用场景**:
- 需要特定的手持跟拍感（轻微晃动但主体清晰）
- 需要特定的运镜轨迹（如环形围绕、推拉节奏）

**操作步骤**:
1. 用手机拍摄一段 3-5 秒的参考视频
2. 上传至 @Reference，选择“动态锚定”模式
3. 在 Prompt 中添加：`运镜风格参考上传视频`

**参考视频拍摄标准**:
- ✅ 手持拍摄，保持轻微自然晃动（完全稳定反而像CG）
- ✅ 运镜轨迹清晰（推/拉/摇/移/跟，选择其一）
- ✅ 时长 3-5 秒（过长无益，AI 只需要运动特征）
- ❌ 快速甩动或剧烈晃动（AI 会学习到错误运动模式）
- ❌ 多段不同运镜拼在一起的视频

**实战案例**:
```
目标: 模拟真人手持手机拍摄的“第一人称探店感”

上传参考视频: 手持手机走过厨房、对准灶台的 5 秒跟拍

Prompt:
【镜头1】特写, 鸡蛋在锅中滑动, 运镜风格参考上传视频, 4K
【镜头2】近景, 铁铲翻面动作, 运镜风格参考上传视频, 4K
```

## 三、组合锚定（最高精度模式）

当需要同时控制光照、主体和运镜时，三种锚定可以组合使用。

### 组合规则

| 锚定组合 | 适用场景 | 参考素材数量 |
|:---|:---|:---|
| 光照 + 主体 | 产品展示类视频（固定机位） | 1 张光照图 + 3 张产品图 = 4 张 |
| 光照 + 动态 | 场景氛围类视频（无特定产品） | 1 张光照图 + 1 段运动视频 |
| 主体 + 动态 | 手持产品测评类视频 | 3 张产品图 + 1 段运动视频 |
| 光照 + 主体 + 动态 | 电影级产品广告（最高精度） | 1 张光照图 + 3 张产品图 + 1 段运动视频 = 4 张 + 1 段 |

### 组合锚定 Prompt 模板

```
【镜头X】[景别], [主体]在[场景]中[动作], 
光照风格参考上传图片1, 
产品外观严格参考上传图片2-4, 
运镜风格参考上传视频1, 
4K画质, Seedance 2.0专用
```

## 四、常见错误与解决方案

| 错误 | 现象 | 解决方案 |
|:---|:---|:---|
| 光照图与场景不兼容 | AI 忽略光照图，输出默认光照 | 选择与 Prompt 场景匹配的光照图（如“厨房”场景选“窗边侧逆光”而非“户外逆光”） |
| 产品图背景干扰 | AI 把产品图的背景也生成了 | 产品图必须纯色背景，或用抠图工具去除背景 |
| 运动视频过长 | AI 学习到复杂运动，输出混乱 | 截取 3-5 秒，单一运镜 |
| 锚定过多 | AI 忽略部分锚定 | 最多 12 个参考，建议 4-6 个效果最佳 |
| 锚定与 Prompt 冲突 | 模型输出不稳定 | Prompt 中的光照/运镜描述必须与锚定素材一致（不要 Prompt 写“硬光”但上传“柔光”参考图） |

## 五、与现有工作流的集成（v4.1）

### 在 SKILL.md 阶段 6 中的调用方式

```
当用户产品属于以下类别时，自动建议启用多模态参考：
- 高客单价产品（$50+）：产品外观一致性直接影响信任度
- 有独特设计的产品：LOGO、特殊造型、纹理需精确还原
- 多镜头视频：4 段镜头需要统一光照风格
```

### 成本估算

| 锚定模式 | 额外准备时间 | 素材要求 |
|:---|:---|:---|
| 仅光照 | 1 分钟（找图） | 1 张图 |
| 仅主体 | 3 分钟（拍三视图） | 3 张图 |
| 仅动态 | 1 分钟（拍短视频） | 1 段 5 秒视频 |
| 全组合 | 5 分钟（拍全套） | 4 张图 + 1 段视频 |

> **投入产出比判断**: 如果该产品预计投放 ≥ $500 广告费，建议全组合锚定。$50-500 建议光照+主体。$50 以下仅文字 Prompt 即可。
