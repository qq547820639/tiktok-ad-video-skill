# A/B 测试矩阵模板 (A/B Testing Matrix) v2.7

> **用途**：为 Skill 提供系统化的多变量测试框架，精准定位爆款要素，避免凭感觉盲测。
> **使用场景**：当需要对比不同钩子类型、脚本结构、原生感策略、社交货币话术、垂直领域信号、生成模式的效果时，使用本模板设计测试方案并记录结果。
> **更新版本**：v2.7 (2026.05) —— 统一版本号，新增社交货币话术 A/B 测试模块、垂直领域信号对比测试、AIGC 合规影响测试、Fast 模式 vs 标准模式成本效益测试。


## 一、测试设计原则

| 原则 | 说明 |
| :--- | :--- |
| **单变量优先** | 每次测试只改变 1 个变量，其他条件完全相同，确保归因清晰。 |
| **样本量要求** | 每个变体至少获得 1000 次展示或 100 次完整观看，数据才有初步参考价值。 |
| **同时段发布** | 不同变体应在相近时段（24h 内）发布到同一账号，减少时间因素干扰。 |
| **同一平台测试** | 不同平台的算法差异巨大，A/B 测试应在单一平台（如 TikTok）完成后再跨平台推广。 |
| **统计显著性** | 简单判断：如果 A 比 B 某项指标高出 30% 以上且样本量过百，可初步判定 A 胜出。 |
| **成本控制** | 测试阶段优先使用 **Fast 模式**（约 60-84 积分/次），确认方向后再升级标准模式。 |
| **合规检查** | 所有测试变体发布时务必勾选对应平台的 **AIGC 标签**，避免合规问题干扰测试结果。 |

## 七、AIGC 合规对测试的影响说明（新增）

| 影响项 | 说明 | 应对措施 |
| :--- | :--- | :--- |
| **限流干扰** | 未勾选 AIGC 标签可能导致变体被限流，测试结果失真 | 所有测试变体发布时强制勾选对应平台的 AI 标签 |
| **标签对完播率的影响** | 部分用户看到 AI 标签后可能提前划走 | 在测试中保持一致：所有变体要么都勾选，要么都不勾选（推荐都勾选） |
| **平台差异** | 不同平台对 AIGC 标签的敏感度不同 | TikTok 影响较小，Meta 影响中等，YouTube 影响较大。测试时记录各平台差异 |
| **测试结论有效性** | 合规状态不一致会导致归因错误 | 确保所有变体在相同合规状态下测试 |
