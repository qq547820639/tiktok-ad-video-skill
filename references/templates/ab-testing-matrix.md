# Thompson Sampling 测试框架 (v4.0-Adaptive)

> **文件定位**: 原 A/B 测试矩阵模板，v4.0 升级为 Thompson Sampling 臂选择与连续收敛判定框架  
> **核心理念**: 在小样本（n < 30）下，用概率分布而非点估计来决策，让探索与利用自动平衡  
> **v4.0 变更**: 放弃简单 A/B 分桶，采用 Beta-Bernoulli 汤普森采样

---

## 一、为什么从 A/B 测试切换到 Thompson Sampling

| 维度 | 传统 A/B 测试 | Thompson Sampling (v4.0) |
|:---|:---|:---|
| 决策方式 | 等到 p < 0.05 | 每次试验选最优概率的臂 |
| 样本效率 | 需要 n ≥ 30+ / 臂 | 5-10 次试验即可显现优势方向 |
| 探索成本 | 固定流量分配，败者组浪费 | 自动收敛，败者组试验次数递减 |
| 适用场景 | 大样本、长时间、稳定环境 | 小样本、快速迭代、非平稳环境 |
| 与贝叶斯系统对接 | 需手动转化 | 直接回写 Beta(α, β) 后验 |

## 三、Thompson Sampling 选择流程

### 每轮执行步骤

1. **从先验中采样**: 对每个臂 j，从 Beta(αⱼ, βⱼ) 中抽取一个随机值 θⱼ
2. **选择最优臂**: 选 θⱼ 最大的臂作为本轮推荐
3. **用户盲选**: 呈现该臂的 Hook 文案给用户，用户选择（是/否偏好）
4. **更新后验**:
   - 若用户选择（成功）: αⱼ += 1
   - 若用户未选择（失败）: βⱼ += 1
5. **记录并重复**: 记录本轮结果，下一轮回到步骤 1

### 代码伪逻辑
```
for each round:
    for each arm j:
        θⱼ = random_sample(Beta(αⱼ, βⱼ))
    
    selected_arm = argmax(θ)
    present_hook_to_user(selected_arm)
    
    if user_chooses:
        α_selected += 1
    else:
        β_selected += 1
```

## 五、连续收敛判定（🆕 v4.0）

> **替代传统 A/B 的“p < 0.05 即停止”规则**

### 收敛标准
当以下条件**同时满足**时，判定该测试已收敛，可停止盲测：

1. **最佳臂稳定**: 连续 5 轮中，同一臂被选中 ≥ 4 次
2. **后验集中**: 最佳臂的 P(Viral) 95% 可信区间宽度 < 0.30（即 [上限 - 下限] < 0.30）
3. **样本量下限**: 最佳臂的 (α + β) ≥ 5

### 收敛后动作
- 最佳臂的 Hook 类型 → 正式投放
- 最佳臂的最终 P(Viral) & CI → 写入该品类-市场的贝叶斯汇总表
- 次优臂的 Hook 类型 → 降级为备选，在新一轮冷启动时降低初始权重
- 若最佳臂的最终 P(Viral) < 0.55 → 所有臂均不合格，启动 Hook 重新设计

### 非收敛判定（提前终止条件）
- 所有臂的 P(Viral) 均 < 0.40 → 放弃本轮 Hook 方向，重新分析竞品拆解
- 10 轮后仍未收敛 → 不强制继续，当前最佳臂直接投放，标记为“未收敛-探索”

## 六、与传统 A/B 测试的对照迁移

| 传统 A/B 概念 | Thompson Sampling 替代 |
|:---|:---|
| 对照组 vs 实验组 | 所有臂平等竞争，收敛自动淘汰 |
| 样本量计算器 | 不需要，收敛判定替代固定 n |
| p-value 显著性 | 后验概率 P(Best Arm = Arm A) > 0.95 |
| 最小可检测效应量 | 可信区间宽度 < 0.30 |
| 测试周期固定 | 连续收敛判定，灵活终止 |
| AA 测试（系统校验）| 使用 Beta(1,1) 先验做一轮空跑，验证采样逻辑 |

## 七、示例：铸铁锅 · 3 臂 Thompson Sampling 完整记录

### 初始设置
- 产品: 铸铁元宝锅
- 市场: 美国
- 臂定义:
  - Arm A: 冲突型 - "这不是魔术，这是物理！" | α₀=3, β₀=1（基于 S-001 成功先验）
  - Arm B: 好奇缺口型 - "用了 20 年铁锅，今天才知道开锅是错的" | α₀=1, β₀=1
  - Arm C: 价格冲突型 - "$29.9 的价格，$299 的质感" | α₀=1, β₀=1

### 试验记录
| 轮次 | 选中臂 | θᴀ | θʙ | θᴄ | 用户选择 | Arm A (α,β) | Arm B (α,β) | Arm C (α,β) |
|:---|:---|:---|:---|:---|:---|:---|:---|:---|
| 1 | A | 0.71 | 0.45 | 0.52 | 是 | (4,1) | (1,1) | (1,1) |
| 2 | A | 0.79 | 0.38 | 0.61 | 是 | (5,1) | (1,1) | (1,1) |
| 3 | A | 0.83 | 0.55 | 0.42 | 否 | (5,2) | (1,1) | (1,1) |
| 4 | C | 0.68 | 0.33 | 0.73 | 是 | (5,2) | (1,1) | (2,1) |
| 5 | A | 0.76 | 0.41 | 0.59 | 是 | (6,2) | (1,1) | (2,1) |

### 收敛判定（第 5 轮后）
- 最近 5 轮中 Arm A 被选中 4 次 → ✅ 条件1
- Arm A 后验 P(Viral) = 6/(6+2) = 0.75, 95% CI: [0.40, 0.96] → 宽度 0.56 → ❌ 条件2 未满足
- 样本量 n = 8 → ✅ 条件3
- **结论**: 未完全收敛，继续测试 2-3 轮

### 最终结果（第 8 轮后，模拟）
- Arm A: α=8, β=2, P(Viral)=0.80, 95% CI [0.49, 0.96] → 宽度 0.47（接近收敛边缘）
- 决策: Arm A 投放，但因 CI 仍宽，标记为“高置信度但区间未收紧，小预算测试”
