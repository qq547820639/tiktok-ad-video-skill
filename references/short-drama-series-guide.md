# 微短剧系列化脚本模板库 (Short Drama Series Guide) v1.0

> **用途**：为需要系列化、故事化运营的品类提供多集微短剧设计模板。将已验证的“末日生存”等成功模式沉淀为可复用的方法论。
> **适用场景**：用户希望用持续性的故事角色（而非一次性产品展示）来建立粉丝粘性和品牌认知。
> **创建版本**：v1.0 (2026.05) —— 初始版本，整合多行业实战经验。


## 一、系列化 vs 单集视频的核心差异

| 维度 | 单集广告/软广 | 微短剧系列 |
| :--- | :--- | :--- |
| **内容目标** | 单次转化 | 粉丝积累 + 品牌认知 + 持续转化 |
| **用户关系** | 一次性观看 | 追更、期待下一集 |
| **产品角色** | 主角/被推销对象 | 自然而然的“剧情道具” |
| **内容周期** | 单条视频的生命周期 | 系列的生命周期（数周甚至数月） |
| **算法友好度** | 单条爆款依赖运气 | 系列化内容更容易被算法识别为“优质连续内容”，获得持续推荐 |
| **适合品类** | 几乎所有 | 强过程展示、高复购率、粉丝经济属性强的品类 |


## 二、系列化结构设计：四大核心模块

### 2.1 角色设定

> 角色是系列化的灵魂。成功的微短剧系列通常只有 1-2 个核心角色。

| 角色类型 | 特征描述 | 适用品类 | 示例 |
| :--- | :--- | :--- | :--- |
| **生存专家** | 在一个极端环境中利用有限资源生存 | 工具、厨具、户外用品 | 末日生存者、荒野求生者 |
| **生活小白** | 在某个领域从零开始学习，产品作为“学习工具”出现 | 厨具、美妆、DIY工具 | 刚搬出父母家的年轻人、第一次做饭的新手 |
| **职业达人** | 利用专业技能展示产品，观众通过角色获得“行业秘密” | 清洁工具、专业设备 | 专业清洁工、厨师、机械师 |
| **改造狂人** | 对旧物/旧空间进行低成本改造 | 家居收纳、装修工具 | 租房改造者、旧物翻新达人 |

### 2.2 世界观构建

> 世界观决定了系列的独特性和记忆点。

| 世界观类型 | 核心设定 | 适合品类 | 成功案例特征 |
| :--- | :--- | :--- | :--- |
| **末日生存** | 资源匮乏环境，产品成为“生存关键工具” | 多功能工具、耐用厨具、储能设备 | 强调产品在极端环境下的可靠性 |
| **普通日常** | 真实生活场景，产品作为“提升生活质量”的自然组成部分 | 厨具、收纳、美妆 | 贴近生活，容易引发共鸣 |
| **行业揭秘** | 以“业内人士”视角揭露行业内幕，产品体现专业标准 | 清洁、维修、餐饮设备 | 教育+娱乐，观众获得“内行知识” |
| **时间循环** | 同一场景反复出现，每次用不同产品/方法解决 | 清洁用品、厨房工具 | 天然的“Before/After”对比结构 |

### 2.3 单集结构（1分钟左右）

| 段落 | 时长 | 内容 | 功能 |
| :--- | :--- | :--- | :--- |
| **开头钩子** | 0-5s | 上集回顾 / 本集悬念 / 角色困境 | 留住老观众，吸引新观众 |
| **困境展开** | 5-20s | 角色面对的具体问题，尝试用旧方法解决失败 | 制造冲突，让产品登场有“必要性” |
| **产品登场** | 20-40s | 产品作为“解决方案”自然出现，过程展示+效果验证 | 软性植入，观众感受到“原来如此” |
| **收尾钩子** | 40-60s | 角色状态改变 + 下集预告 / 开放式问题 | 驱动追更、评论互动 |

### 2.4 长效钩子设计

> 系列化的核心是让用户“想看下一集”。

| 钩子类型 | 实现方式 | 示例 |
| :--- | :--- | :--- |
| **未完成结局** | 结尾展示一个未解决的问题 | “但这个东西还有一个致命缺点...” |
| **角色成长线** | 观众跟随角色从“小白”变成“达人” | “第一集连锅都不会用，第五集能做出满汉全席” |
| **观众参与决策** | 让评论区投票决定下一集的内容方向 | “下一集你们想看我用它做什么菜？评论区告诉我” |
| **隐藏彩蛋** | 每集埋一个只有忠实观众才能发现的细节 | 背景中出现上一集用过的某个物品 |


## 三、品类适配指南

| 品类 | 推荐剧情类型 | 推荐角色类型 | 核心卖点呈现方式 |
| :--- | :--- | :--- | :--- |
| **锅具/厨房用品** | 末日生存、普通日常、生活小白成长 | 生存专家、生活小白 | “没有这个锅，他不可能做出这顿饭” |
| **清洁用品** | 行业揭秘（清洁工视角）、改造狂人 | 职业达人、改造狂人 | “专业清洁工用这个工具，5分钟搞定” |
| **收纳用品** | 改造狂人（租房改造）、普通日常 | 改造狂人、生活小白 | “这个400尺的公寓，一个架子就解决了所有收纳” |
| **美妆工具** | 生活小白成长、职业达人 | 生活小白、职业达人 | “从手残到能出门，只差这个工具” |
| **3C/家电** | 末日生存（储能）、行业揭秘（效率工具） | 生存专家、职业达人 | “停电72小时后，只有这个东西还在运转” |
| **户外用品** | 末日生存、荒野求生 | 生存专家 | “带着这个东西，我一个人在野外住了7天” |


## 四、转化点植入时机

> 微短剧的转化不能像直给型广告那样“最后3秒喊价”，需要在系列的不同阶段设置不同的转化策略。

| 系列阶段 | 转化策略 | 示例 |
| :--- | :--- | :--- |
| **第1集** | **零转化**，纯粹吸引关注 | 不提及任何购买信息，只展示角色的困境和产品登场 |
| **第2-3集** | **评论区软引导** | 回应评论时提及产品，评论区置顶“Link in bio” |
| **第4-5集** | **bio链接显性化** | 个人主页简介更新为“The tools I use: [链接]” |
| **第6集以后** | **专属系列折扣** | 为追更粉丝提供专属折扣码，强化“系列粉丝福利”感 |

**转化形式**：
- **自然提及型**：角色在剧情中随口说“还好有这个XX”，画面不刻意特写产品。
- **过程展示型**：产品使用过程本身就是内容核心（如锅具的烹饪过程），观众自发追问“这是什么”。
- **评论区驱动型**：在评论区发起“你觉得这个工具怎么样？”的讨论，引导用户深度参与。
- **彩蛋转化型**：在每集结尾展示一个“幕后花絮”或“使用小技巧”，将关注点自然导向产品。


## 五、与 Skill 工作流的集成

### 阶段 0.5 触发条件

当用户明确要求“做一个系列”或提到“微短剧”时，AI 自动启用本指南。

### 输出内容

1.  **角色设定卡**：角色名称、性格特征、代表动作/口头禅。
2.  **世界观概述**：一句话说明系列背景。
3.  **3 集内容大纲**：每集的核心冲突、产品使用场景、结尾悬念。
4.  **转化点规划**：每集的转化策略和强度。

### 输出模板

```
【微短剧系列策划 · [产品名称]】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎭 角色设定：
- 角色名：[名称]
- 性格特征：[描述]
- 口头禅/标志动作：[描述]

🌍 世界观：
- [一句话说明]

📺 3 集大纲：
第1集：[核心冲突] → [产品登场] → [悬念]
第2集：[新挑战] → [产品深入使用] → [悬念]
第3集：[高潮冲突] → [产品完全展现价值] → [阶段性收尾+下季预告]

🛒 转化规划：
第1集：零转化
第2-3集：评论区软引导
第4集起：bio链接+专属折扣
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```


## 六、案例参考

### 成功案例：锅具系列 · 末日生存主题

| 维度 | 内容 |
| :--- | :--- |
| **角色** | 一个在“末日世界”独自生存的年轻人 |
| **世界观** | 资源极度匮乏，所有工具都必须是多功能的、极端耐用的 |
| **产品** | 铸铁元宝锅 |
| **第1集** | 主角在废墟中找到一口被遗弃的锅。简单清洗后，用少量食材做出了一顿“末日美食”。结尾：远处传来奇怪的声音。 |
| **第2集** | 主角用这口锅“一锅出”了煎蛋+炒菜+煮汤，展示其多功能性。结尾：另一个幸存者出现了。 |
| **第3集** | 两人用这口锅合作做了一顿“末日盛宴”，展示锅的容量和耐用性。结尾：两人决定结伴前行。 |
| **转化效果** | 前三集零广告，评论区大量自发追问“这是什么锅”。第四集起在bio中挂链接，转化率比直给型广告高3倍。 |


## 七、系列化自检清单

- [ ] **角色是否有记忆点**？（观众能记住TA吗？）
- [ ] **世界观是否有独特性**？（与其他系列区分开）
- [ ] **每一集是否都有独立的冲突和解决**？（单集可独立观看）
- [ ] **每集结尾是否都有“想看下一集”的冲动**？
- [ ] **产品是否始终是“剧情道具”而非“广告主角”**？
- [ ] **转化节奏是否合理**？（第1集绝不硬推）
- [ ] **评论区是否准备好引导话术**？
