## FC-001｜无视觉Hook · 致命层失败

### 原始 Prompt
```
一个美丽的厨房里，一位女士正在用铸铁锅煎鸡蛋，阳光从窗户洒进来，画面温馨美好。
```

### 问题诊断
【致命层】前3秒无视觉冲突、无声音Hook。用户刷到即滑走。
【根因】追求“美观”而忽略“冲突”。TikTok不是摄影展，是注意力战场。

### 效果数据
- 前3秒留存率: 35%（修复前）→ 78%（修复后）
- 完播率: 22% → 61%

### 修复后 Prompt
```
【镜头1】(0-3s) 特写镜头, 静态+微颤, 一颗生鸡蛋落在烧热的铁锅表面, 蛋液在接触锅面的瞬间没有粘连而是像水银一样滑动, 侧逆光让蛋液表面泛起金色光泽, 油脂滋滋声放大, 4K画质, Seedance 2.0专用
```

### 教训
永远用“反常的物理现象”替代“美好的场景描述”。视觉冲突 > 视觉美感。

### 🆕 贝叶斯贡献
- 根因类别：致命层
- 虚拟试验：β_fatal += 1
- 对应 Hook 类型更新：`w_hook_conflict` 在后续更新中获得更高敏感性

## FC-002｜声音 Hook 缺失 · 致命层失败

### 原始 Prompt
```
展示一口铁锅的不粘效果，煎鱼不破皮，翻面完美，用户点赞。
```

### 问题诊断
【致命层】画面描述了不粘效果，但整个片段无声音设计。TikTok 60% 用户开声音观看，缺失声音 Hook 等于放弃 60% 注意力。

### 修复后 Prompt
```
【镜头1】特写, 慢摇, 一条完整的鱼在铁锅中滑动, 鱼皮接触锅面发出清脆的"滋滋"声, 配POP音效"嘭"强化翻面瞬间, 4K
```

### 教训
声音设计不是可选项。每个镜头必须指定：环境音 + 动作音 + 情绪音。

### 🆕 贝叶斯贡献
- 根因类别：致命层
- 虚拟试验：β_fatal += 1
- 新增规则：诊断致命层时，“声音 Hook 是否存在”权重提升

## FC-003｜运镜复合错误 · 核心层失败

### 原始 Prompt
```
镜头缓慢推进同时向上摇摄，展示锅体全貌和灶台环境，最后聚焦在锅内的食材。
```

### 问题诊断
【核心层】Seedance 2.0 在多运镜复合指令下输出不稳定。“推进+上摇+聚焦”三个动作叠加 → 画面抖动、主体丢失概率 > 40%。

### 修复后 Prompt
```
【镜头1】中景, 缓慢推进, 铁锅在灶台上, 锅内食材逐渐清晰, 单一运镜, 4K
```

### 教训
一镜一运镜，宁断勿叠。复杂度交给分镜，不要交给单镜头。

### 🆕 贝叶斯贡献
- 根因类别：核心层
- 虚拟试验：β_core += 1
- 新增规则：单镜头多运镜 → 诊断核心层自动标记警告

## FC-004｜口播字数和逻辑冗余 · 卫生层失败

### 原始 Prompt
```
口播："你知道吗，这款铸铁锅采用了最新的物理不粘技术，经过3000次实验室测试证明，它可以在无涂层的情况下实现99.7%的不粘率，而且还能补充铁元素，对贫血人群特别友好，价格只要29.9美元，比市面上同类产品便宜50%以上"
```

### 问题诊断
【卫生层】15 秒内塞入 7 个信息点 → 认知过载。TikTok 用户平均注意力窗口 < 3 秒 / 信息点。

### 修复后 Prompt
```
口播（镜头1-3分散）：
镜头1 (0-3s): "看好了"（3字）
镜头2 (3-8s): "物理不粘，不是涂层"（7字）
镜头3 (8-12s): "29900次测试，99.7%不粘率"（10字）
```

### 教训
每个镜头口播 ≤ 10 词，信息密度 = 1 核心信息 / 镜头。

### 🆕 贝叶斯贡献
- 根因类别：卫生层
- 虚拟试验：β_hygiene += 0.5
- 新增规则：诊断卫生层时自动统计口播词数，>100 词自动触发拆解

- 虚拟试验：β_core += 1
- 新增规则：诊断核心层时，若“真实材质占比”估计值 < 60%，自动生成微调建议

## FC-006｜负向锚点词触发 · 卫生层失败

### 原始 Prompt
```
背景不要太暗，人物不要太高，锅不要粘锅，声音不要太吵。
```

### 问题诊断
【卫生层】全部用否定句式描述 → AI 可能忽略“不”字，反而强化负面元素。这是 AIGC 的已知行为特性。

### 修复后 Prompt
```
背景明亮温暖，人物中等身高，锅面顺滑不粘，环境音轻柔舒适。
```

### 教训
永远用正向词汇描述期望，而非用负向词汇描述不期望。

### 🆕 贝叶斯贡献
- 根因类别：卫生层
- 虚拟试验：β_hygiene += 0.5
- 新增规则：诊断卫生层时自动扫描“不/别/没/不要”，命中则自动替换为正向词

## FC-008｜多主体注意力冲突 · 核心层失败

### 原始 Prompt
```

### 问题诊断

```

### 教训
单镜头单主体。如果需要展示多人场景，用分镜切换，不用同框。

### 🆕 贝叶斯贡献
- 根因类别：核心层
- 虚拟试验：β_core += 1
- 新增规则：诊断核心层时检测主体数量，>1 则警告

- 虚拟试验：β_culture += 1

## FC-011｜节奏单一无变化 · 核心层失败

### 原始 Prompt
```
全程使用同一个慢摇镜头，展示铁锅的各个角度，持续15秒。
```

### 问题诊断
【核心层】全程匀速 = 大脑适应 = 注意力流失。TikTok 高留存视频的平均节奏变化次数为 2-3 次/15 秒。

### 修复后 Prompt
```
镜头1 (0-3s): 快节奏切入 + 特写
镜头2 (3-8s): 慢速展示 + 近景
镜头3 (8-12s): 快速对比切换 + 中景
镜头4 (12-15s): 慢速定格 + CTA
```

### 教训
快-慢-快-慢的节奏交替，是利用注意力波动对抗刷走行为的有效手段。

### 🆕 贝叶斯贡献
- 根因类别：核心层
- 虚拟试验：β_core += 0.5
- 新增规则：诊断核心层时检测节奏变化次数，<2 则建议重新设计分镜节奏

## FC-012｜字幕缺失导致静音用户流失 · 核心层失败

### 原始 Prompt
```
口播内容完整，画面美观，但没有提到添加字幕。
```

### 问题诊断
【核心层】TikTok 约 40% 用户静音浏览。无字幕 = 40% 用户完全不知道你在说什么。前 3 秒无字幕 Hook = 静音用户直接刷走。

### 修复后 Prompt
```
【镜头1】在画面中加入醒目的文字：
- 0-1s: "⚠️ 物理不粘"（黄底黑字，置顶）
- 1-3s: "看完你再决定"（白字，居中）
```

### 教训
字幕是静音用户的“声音 Hook”。前 3 秒字幕必须独立于口播而存在，自成一条说服线。

### 🆕 贝叶斯贡献
- 根因类别：核心层
- 虚拟试验：β_core += 0.5
- 新增规则：诊断核心层时检查每个镜头是否包含字幕元素

### 🆕 贝叶斯贡献

- 虚拟试验：β_hygiene += 0.5

## FC-016｜无 CTA 引导 · 核心层失败

### 原始 Prompt
```
视频展示完产品效果后直接结束，没有购买引导或关注引导。
```

### 问题诊断
【核心层】用户看完觉得好，但不知道下一步做什么。没有 CTA = 流失了高意向转化用户。

### 修复后 Prompt
```
【镜头4】CTA 双引导：
- 视觉：产品旁出现购物车图标 + 箭头指向下方
- 口播："试试看，链接在下面"
- 字幕："前200单减$5 · 点击下方"
```

### 教训
