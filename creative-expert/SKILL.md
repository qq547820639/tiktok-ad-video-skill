## 一、阶段3：Hook盲测（Thompson Sampling）

### 执行规则

1. 从 `core-knowledge.md` §一的8类Hook中，基于当前品类的Beta先验采样θ值
2. 若该品类无历史数据，使用Beta(1,1)无信息先验
3. 若该品类非消费品（如技术/SaaS），Hook权重独立维护，不污染消费品先验
4. 输出3个选项：A（θ最高）、B（θ次高）、C（随机探索）

### 默认选择策略

- 若编排者传递了用户选择 → 以用户选择为准
- 若编排者标注了"默认选择" → 自动采用θ值最高的臂（臂A）
- **无论用户是否选择，系统必须继续执行阶段4-5-6-6.5**

### 输出格式

```

|:---|:---|:---|:---|

| C | [类型] | [θ值] | "[文案]" |

**最终采用: 臂[A/B/C]**（[用户选择/默认选择]）
```

## 二、阶段4：15秒脚本生成

### 执行规则

1. 基于选定的Hook，生成标准格式脚本
2. 格式必须严格遵循：`镜头编号 | 时间 | 景别·运镜 | 画面描述 | 口播`
3. **必须严格遵守15秒×4镜头的标准结构**（0-3s / 3-8s / 8-12s / 12-15s）
4. **禁止**自创7镜头45秒等非标准格式。若产品需要更多展示，通过分镜切换在4镜头内完成
5. 必须包含复播彩蛋（镜头4最后1.5秒的开放循环）
6. 口播中必须使用 `/` 停顿标记（每12字左右插入）

### 输出格式

```
## 【阶段4：15秒脚本】

**文案**: [≤5字，情感锚点]

| 镜头 | 时间 | 景别·运镜 | 画面描述（含声音设计） | 口播 |
|:---|:---|:---|:---|:---|
| 1 | 0-3s | [景别]·[运镜] | [视觉冲突+声音Hook] | "[口播]" |
| 2 | 3-8s | [景别]·[运镜] | [核心卖点视觉化+声音层] | "[口播]" |
| 3 | 8-12s | [景别]·[运镜] | [使用场景+结果对比+声音层] | "[口播]" |
| 4 | 12-15s | [景别]·[运镜] | [CTA+复播彩蛋+声音层] | "[口播]" |

**声音层设计**:
- 环境音: [贯穿全片的环境声音描述]
- 动作音: [各镜头关键动作的打击音列表]
- 情绪音: [BGM情绪描述及变化节点]
```
