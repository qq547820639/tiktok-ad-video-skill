## ⚡ 30秒极简加载指南

1. 复制 `SKILL.md` 全部内容
2. 粘贴到 AI 对话框（DeepSeek 用户放入 System Prompt 设置区域）

4. 发送后，输入你的产品信息，开始生成视频

如需深度扩展知识（完整 Hook 变体库、动态权重引擎、失败案例库等），可额外加载 `references/` 目录下的 6 个文件。

### 第二步：输入产品信息

向加载了 Skill 的 AI 助手发送你的产品信息：

> “我卖 [产品名称]，核心卖点是 [一句话描述]，价格在 [价格区间]，目标客户是 [人群描述]。”

**示例**：

> “我卖铸铁元宝锅，无涂层物理不粘，价格 $29.9，目标客户是 25-45 岁家庭烹饪用户。”

### 第三步：Hook 盲测（Thompson Sampling · 支持默认选择）

Skill 会输出 **3 个爆款 Hook 文案选项**（例如 A/B/C），请凭直觉选择最能吸引你的一个。

**v4.2 增强**：如果你不选择，系统会自动采用 θ 值最高的 Hook 继续执行，不会卡住流程。

### 第四步：获取 15 秒脚本

Skill 会根据你的选择和产品品类，输出标准格式的 15 秒多镜头脚本。

### 第五步：贝叶斯爆款预测（强制执行 · 积分风控）

**v4.2 核心升级**：诊断摘要**每次强制输出**，不依赖用户反馈。

提交生成**之前**，Skill 自动完成以下诊断：
- **致命层**：Hook 是否有视觉冲突？是否有声音Hook？ → 任一不通过，**禁止提交**
- **核心层**：垂直信号是否双出现？原生感是否≥60%？ → 不通过则警告+输出修复版
- **卫生层**：是否含负向锚点词？口播是否标注停顿标记？ → 自动修正
- **🆕 贝叶斯概率输出（强制）**：
  - P(Viral) = α / (α + β)
  - 95% Bootstrap CI (n=1000)
  - 校准状态标注（基于先验 / 基于反馈 / 基于投放数据）
  - P(Viral) < 0.55 → 建议重新设计 Hook
  - P(Viral) ∈ [0.55, 0.75] → 小预算测试
  - P(Viral) > 0.75 → 正式投放

### 第六步：获取 Seedance 2.0 原子化提示词（强制执行）

**v4.2 核心升级**：无论诊断结果如何，Skill **必须输出**可直接粘贴到即梦 AI 的原子化分段提示词。

- **阶梯式算力分配**：镜头1（黄金Hook）用 Standard 模式，镜头2-4 用 Fast 模式。
- **声音层独立规范**：每段 Prompt 包含环境音+动作音+情绪音。
- **多模态参考锚定**：可选，调用 `multimodal-reference-guide.md`。

### 🆕 第七步：内部自评迭代（自动执行）

**v4.2 新增**：Skill 在输出提示词后，自动进行一次内部自评：

1. 重新检查三层诊断
2. 若发现问题 → **在当前回复中直接输出修正版**
3. 重新计算 P(Viral) 和 CI
4. 最多迭代 3 轮，确保输出质量
5. 输出自评历史（修正了哪些问题、P(Viral) 变化）

### 第八步：生成视频与数据回传

1. 将**修正后的分段提示词**逐段粘贴到即梦 AI，分别生成每个镜头。
2. 若某段翻车：**局部重生成**，不推翻全部。
3. 投放后第 7 天回到对话中提供数据，触发权重更新。
