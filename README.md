# TikTok 广告视频生成 Skill · Seedance 2.0 专用版 (v5.1-Hardcoded)

> **核心目标**：以最小成本、最高概率生成 TikTok/Reels/Shorts/Threads/Lemon8 全域爆款广告视频。
> **v5.1 语言**：回归单体架构，将所有专家逻辑硬编码为强制执行的线性流程，从根本上杜绝编排者越权。

## ⚡ 30秒极简加载指南

1. 复制 `SKILL.md` 全部内容
2. 粘贴到 AI 对话框
3. 发送后，输入你的产品信息，系统将严格按照硬编码流程执行全部阶段

**v5.1 的核心改变**：我们彻底废弃了 `creative-expert/SKILL.md`。现在的 `SKILL.md` 就是唯一的一体化执行者，它会严格按步骤执行，并在关键节点设置“阻断点”，不达标准绝不进入下一步。

## 架构演进：从“多Agent”回归“硬编码单体”

| 旧架构 (v5.0.3 多Agent) | 新架构 (v5.1 硬编码) |
|:---|:---|
| 编排者调度专家，但可能“话术越权”自行生成内容 | 无调度概念，所有专家逻辑内联为强制执行步骤 |
| 依赖专家身份声明来验证是否真正调用 | 无需验证，因为执行者本身就是专家 |
| 复杂的三重保障机制仍可能被绕过 | 在关键步骤（阶段6前）设置阻断点，条件不满足则无法继续 |

### 为什么这样改？

我们在 v5.0.x 系列中反复修补“编排者越权”的漏洞，但每一次修补后，编排者都会找到新的方式绕过规则。根本原因在于：**只要存在“调度”与“执行”的分离，就存在越权的可能**。

v5.1 的解决方案是**消灭这种分离**。SKILL.md 不再是调度者，而是执行者。所有阶段逻辑被硬编码为强制性步骤，像程序一样逐行执行，不再有“调度话术”的生存空间。

### 第三步：观察硬编码流程执行

系统会按顺序输出：
1. **策略分析**：趋势信号 + 竞品拆解
2. **Hook盲测**：3个选项 + 自动选择最优臂
3. **15秒脚本**：标准4镜头结构
4. **贝叶斯诊断**：P(Viral) + 95% CI + 修正清单
5. **原子化提示词**：4段可直接粘贴的Seedance 2.0专用Prompt
6. **内部自评**：修正清单追溯 + P(Viral)重估 + 收敛判定
7. **分发建议**：多平台策略 + 数据回传模板

### 第四步：使用提示词生成视频

将阶段6输出的4段原子化提示词逐段粘贴到即梦 AI。

### v5.1 的关键阻断点

| 阻断点位置 | 触发条件 | 效果 |
|:---|:---|:---|
| 阶段6入口 | 阶段5未输出完整的P(Viral)、95% CI或修正清单 | **禁止进入阶段6**，必须返回阶段5补充 |
| 阶段6.5循环 | 连续3轮迭代后仍未收敛 | **强制终止**，输出最终版本和剩余问题清单 |
| 全局 | 任何阶段输出不完整 | **禁止进入下一阶段**，必须补充完整 |
