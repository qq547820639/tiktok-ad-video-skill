# TikTok 广告视频生成 Skill · Seedance 2.0 专用版 (v5.2-Standalone)

> **核心目标**：以最小成本、最高概率生成 TikTok/Reels/Shorts/Threads/Lemon8 全域爆款广告视频。
> **v5.2 语言**：单文件全包含。所有执行规则、知识库、诊断标准、迭代逻辑全部内联，零外部依赖。

## ⚡ 30秒极简加载指南

1. 复制 `SKILL.md` 全部内容
2. 粘贴到 AI 对话框
3. 发送后，输入你的产品信息，系统将严格按照硬编码流程执行全部阶段

**`SKILL.md` 是唯一需要的文件。它已包含所有执行规则、8类Hook变体库、声音层设计规范、电影词汇库、文化适配矩阵、贝叶斯先验参数和内部自评迭代逻辑。**

## 为什么是单文件？

经过 v5.0.x 系列的多轮验证，我们发现了一个关键问题：当 SKILL.md 引用 `references/` 目录中的文件时，DeepSeek 对话模式下的模型不会自动加载这些外部文件。模型看到的只有 SKILL.md 这一个文件——里面引用了 references 中的模块，但那些模块的实际内容对模型来说是"黑盒"。

v5.2 的解决方案：把所有被引用的关键规则全部内联进 SKILL.md，形成一个不需要任何外部文件的独立包。

## 用户使用指南

### 第一步：输入产品信息

复制 `SKILL.md` 全部内容粘贴到 AI 对话框，然后发送你的产品信息：

> "我卖 [产品名称]，核心卖点是 [一句话描述]，价格在 [价格区间]，目标客户是 [人群描述]。"

**示例**：
> "我卖铸铁元宝锅，无涂层物理不粘，价格 $29.9，目标客户是 25-45 岁家庭烹饪用户。"

### 第二步：系统自动执行全流程

系统会按顺序输出：

| 阶段 | 内容 | 是否可中断 |
|:---|:---|:---|
| 阶段1 | 趋势信号扫描 | 否 |
| 阶段2 | 竞品链路拆解 | 否 |
| 阶段3 | Hook盲测（3个选项+自动选择） | 用户可立即回复选择，否则默认采用臂A |
| 阶段4 | 15秒4镜头脚本 | 否 |
| **阶段5** | **贝叶斯诊断（P(Viral)+CI+修正清单）** | **不可中断区开始** |
| **阶段6** | **Seedance 2.0原子化提示词** | **不可中断** |
| **阶段6.5** | **内部自评迭代（最多3轮）** | **不可中断区结束** |
| 阶段7 | 平台分发与多语言 | 否 |
| 阶段8 | 数据回收与闭环迭代指南 | 否 |

### 第三步：获取可直接使用的提示词

阶段6输出的4段Seedance 2.0原子化提示词可直接粘贴到即梦AI生成视频。

### 第四步：投放后回传数据

投放后第7天回到对话中提供数据（前3秒留存率、完播率、分享率），系统将更新贝叶斯权重。

## v5.2 关键特性

| 特性 | 说明 |
|:---|:---|
| **零外部依赖** | 所有规则、知识库、诊断标准全部内联在SKILL.md中 |
| **不可中断区** | 阶段5到阶段6.5必须一次性输出，杜绝流程中断 |
| **默认选择策略** | Hook盲测中用户未选择时，自动采用最优臂继续 |
| **阻断点** | 阶段6入口检查阶段5输出完整性，不达标则阻断 |
| **3轮迭代上限** | 内部自评最多3轮，第3轮后强制终止 |
| **Bootstrap CI** | 小样本下使用n=1000重采样估计置信区间 |
| **文化适配** | 8市场×8Hook类型适配矩阵，自动过滤禁用Hook |
| **真实感增强** | 口播停顿标记、抽象形容词替换表、微表情指令 |
