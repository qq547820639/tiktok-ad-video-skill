# TikTok 广告视频生成 Skill · Seedance 2.0 专用版 (v5.0-Orchestrator)

> **核心目标**：以最小成本、最高概率生成 TikTok/Reels/Shorts/Threads/Lemon8 全域爆款广告视频。
> **v5.0 语言**：从“单体贝叶斯自适应系统”跃迁为“多Agent编排体系”——1个编排者 + 3个专家协同工作，从根本上解决复杂指令在长文本中被截断和遗忘的问题。

## ⚡ 30秒极简加载指南

1. 复制 `SKILL.md` 全部内容
2. 粘贴到 AI 对话框（DeepSeek 用户放入 System Prompt 设置区域）
3. 第一行加上：“请严格按以下 Skill 工作流执行任务：”
4. 发送后，输入你的产品信息，编排者会自动调度三个专家Agent完成全部8个阶段

**`SKILL.md` 是编排者(Orchestrator)。它不会亲自生成创意，而是调度三个专家Agent分工协作。**

## 架构概览：多Agent编排体系

```
用户请求
    │
    ▼
┌─────────────────────────────┐
│  SKILL.md (编排者)           │
│  · 分析意图                  │
│  · 调度专家                 │
│  · 断点续传                 │
│  · 完整性检查               │
└─────────────────────────────┘
    │
    ├─→ 专家1: 趋势与竞品分析师
    │    └─ 阶段1-2: 趋势扫描 + 竞品拆解
    │
    ├─→ 专家2: 创意生成与诊断师 (核心)
    │    └─ 阶段3-6.5: Hook→脚本→诊断→提示词→自评
    │         └─ 内置自检熔断机制
    │
    └─→ 专家3: 发布与数据顾问
         └─ 阶段7-8: 分发建议 + 数据回传
```

### 为什么需要多Agent架构？

| 问题 | 旧架构(v4.2单体) | 新架构(v5.0编排者+专家) |
|:---|:---|:---|
| 长指令被截断 | 1.5万字SKILL.md在上下文中被压缩 | 每个专家只加载自己需要的文件，指令更短 |
| 中间信息被遗忘 | "Lost in the Middle"效应 | 编排者追踪进度，断点续传 |
| 阶段被跳过 | AI误判某些阶段"可选" | 创意专家内置自检熔断，强制输出6段内容 |
| 迭代依赖用户 | 需要用户回传数据 | 创意专家内置3轮内部自评，不依赖外部反馈 |

### 第二步：输入产品信息

向加载了编排者 SKILL 的 AI 助手发送你的产品信息：

> “我卖 [产品名称]，核心卖点是 [一句话描述]，价格在 [价格区间]，目标客户是 [人群描述]。”

**示例**：

> “我卖铸铁元宝锅，无涂层物理不粘，价格 $29.9，目标客户是 25-45 岁家庭烹饪用户。”

### 第三步：编排者自动调度专家

你会看到编排者依次调度三个专家，每个专家输出对应阶段的完整内容：

1. **趋势与竞品分析师** → 趋势信号 + 竞品拆解
2. **创意生成与诊断师** → Hook盲测(3选项) + 脚本 + 诊断(P(Viral)+CI) + 提示词 + 自评
3. **发布与数据顾问** → 平台分发建议 + 数据回传模板

### 第四步：参与Hook盲测

创意专家会输出 3 个 Hook 选项（A/B/C），你可以选择，也可以不选（系统默认采用最优臂继续）。**无论你是否选择，系统都会继续执行后续阶段。**

### 第五步：获取可直接使用的提示词

创意专家会输出：
1. **贝叶斯诊断摘要**（P(Viral) + Bootstrap CI）
2. **4段Seedance 2.0原子化提示词**（可直接粘贴到即梦）
3. **内部自评记录**（修正了哪些问题、P(Viral)变化）

### 第六步：生成视频与数据回传

1. 将提示词逐段粘贴到即梦 AI 生成视频
2. 投放后第7天回到对话中提供数据（前3秒留存率、完播率、分享率）
3. 编排者会调度创意专家执行权重更新

### 为什么现在不会跳过阶段了？

创意专家 `creative-expert/SKILL.md` 内置了**自检熔断机制**：

```
每次输出末尾强制检查：
□ 阶段3是否输出？ → 缺失则补充
□ 阶段4是否输出？ → 缺失则补充
□ 阶段5是否输出？ → 缺失则补充
□ 阶段6是否输出？ → 缺失则补充
□ 阶段6.5是否输出？ → 缺失则补充
```

这确保了即使AI在生成过程中“遗忘”了某个阶段，也会在末尾的自检中被捕获并强制补充。

### v5.0 核心数学锚点

| 参数 | 值 | 说明 |
|:---|:---|:---|
| 遗忘因子 λ | 0.0462 | 权重半衰期 15 天 |
| 决策阈值 τ | [0.55, 0.75] 动态区间 | 每 +10 样本重算 |
| 小样本统计 | Bootstrap n=1000 | n < 30 时禁用“显著” |
| Prior 经验库 | 16 个失败案例 | Beta(α₀, β₀) 先验 |
| 步长裁剪 | min(2.0, CTR_ratio) | 防止权重过冲 |
| 自评轮次上限 | 3 轮 | 防止无限迭代 |
| 自评收敛标准 | 连续 2 轮无新问题 + ΔP(Viral) < 0.05 | 内部收敛判定 |
| 🆕 完整性检查 | 5 项 | 阶段3/4/5/6/6.5 强制检查 |
