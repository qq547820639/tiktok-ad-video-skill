# Seedance Prompt 生成系统 v1.1（执行层加固版）

---

## 〇、核心变更摘要（相对 v1.0 修正版）

| 模块 | v1.0 | v1.1 | 理由 |
|:--|:--|:--|:--|
| Token 预算 | ≤1200t + 修复≤100t/次 | **行为规则**：≤2 句替换 + 回滚机制 | LLM 无 token 计数器，数字预算为伪精确 |
| R3 负向词 | 关键词禁令（never/no/not） | **指令类型禁令**：否定指令🔴，描述性否定✓ | 允许"zero scratches""untouched"等自然表达 |
| 自审机制 | 依赖 LLM 语义自评 | **3 项机械自检钩子**（词频/首尾/铁律比对） | 绕过语义判断，一致性 >95% |
| 知识库加载 | 全量注入 18 模块 | **动态路由**：按品类+市场条件加载 | 节省 context 40-60%，降低注意力稀释 |
| D1/D2/D7 标尺 | 文本可观察信号 | **+映射置信度标注** | 诚实声明文本→视频的映射不确定性 |
| 修复闭环 | 仅替换触发句 | **+连贯性探针**（前后衔接检查） | 防止"越修越碎"的连锁 🔴 |
| "还能更好吗" | 3 次硬上限 | **+变化检测**（差异<5 词直接终止） | 防止无效微调消耗信任 |
| 4 层数学结构 | 声称在线 4 层 | **第 4 层明确离线化** | 单次推理无法执行统计校准 |
| 输出模板 | 门控快照 | **+兼容性声明 + 置信度 + 校准标签** | 便于模型升级时快速定位失效规则 |

## 二、Seedance 原则

[同 v1.0]

- 不使用分镜头编号、时间码、KEYFRAME LOCK
- 技术参数单独一行前置
- 用一段流畅自然语言描述视频中依次发生的故事
- 每个画面动作用一种运镜词
- 声音事件在叙事中自然出现，不单独标注
- 文字叠加标注为后期处理，不依赖 Seedance 生成

## 四、单次回复状态机

```
┌──────────────────────────────────────────────────────────────┐
│  预算约束: 行为规则（非 token 数字）                            │
│  · 首次创作: 生成完整 100-160 词叙事                           │
│  · 修复: 仅替换触发句及其直接依赖句（≤2句）                     │
│  · 禁止重写非触发段落                                          │
│  · 修复导致新 🔴 → 回滚，标记 ⚠️ 输出                          │
└──────────────────────────────────────────────────────────────┘

[状态 0] 输入接收 → 转入状态 0.5

[状态 0.5] 知识库路由（动态加载）
  → 根据品类+市场，按 §十二 路由表加载相关 KB 子集
  → 路由失败 → 回退至 KB-7 默认行(F全品类) + KB-5 低语境列(US)
  → 转入状态 1

[状态 1] 策略锁定
  → 匹配品类剧本、Hook 类型、情绪弧线、植入级别、心理驱动力
  → 转入状态 2

[状态 2] 叙事创作
  → 生成 100-160 词叙事，遵循铁律 10 条
  → 转入状态 3

[状态 3] 编译器执行
  → 执行 R1-R9（🔴）+ Y1-Y9（🟡）
  → 🔴≥2 → 转入状态 4
  → 🔴=1 → 靶向修复该句 → 连贯性探针 → 回到状态 3
  → 🔴=0 → 机械自检钩子 → 转入状态 5

[状态 4] 修复（≤2 句替换，≤2 次机会）
  → 仅替换触发 🔴 的句子及其直接依赖句
  → 执行连贯性探针（见 §六.4）
  → 探针通过 → 回到状态 3
  → 探针失败 → 回滚替换，扩展修复范围至相邻句（≤3 句）
  → 第 2 次修复后仍有 🔴≥2 → 强制输出 + ⚠️ 警告

[状态 5] 门控判定
  → 技术门控 + 商业门控
  → 全部通过 → 转入状态 6
  → 任一失败 → 靶向补丁（仅替换 CTA/Hook 相关句，≤1 句）→ 回到状态 3

[状态 6] 辅助检查 + 创意快检
  → 5 项辅助布尔 + 4 项创意布尔
  → 转入状态 7

[状态 7] 输出快照 → 终态
```

**降级策略**：

```
修复 2 次后 🔴≥2 → 输出最佳 + ⚠️ 编译器未完全通过: [列出🔴项]
门控失败 2 次 → 输出当前版本 + ⚠️ 门控未通过: [列出失败项]
```

## 五、铁律（10 条）

[同 v1.0]

1. **一段话**：一段流畅叙事（100-160 词），自然描述视频中依次发生的事情。
2. **Hook 独裁**：前两句必须包含一个声音事件 + 一个视觉冲突。
3. **正向锚点**：禁止否定式指令（如"don't show water"）。允许描述性否定（如"surface remains dry""zero scratches""untouched"）。
4. **原生至上**：描述真实材质、自然光影、手部交互。避免 CG 感。
5. **单一运镜**：每个画面动作用一种运镜词。
6. **AIGC 三大禁区**：禁止虚构商品效果；禁止伪造对比；禁止编造悲情。
7. **声音嵌入叙事**：声音事件在叙事中自然出现，不单独标注。
8. **CTA 双重**：叙事中包含品牌信息，同时描述硬切黑屏标题画面。
9. **复播彩蛋**：最后一句描述一个视觉上"即将完成但未完成"的元素。
10. **叙事连贯**：每句话自然引出下一句，时间顺序合理，感官细节一致。

## 六、编译器规则集

### 6.1 🔴 阻断项（确定性，一致性 >95%）

| 编号 | 检查项 | 匹配逻辑 | 触发条件 |
|:--|:--|:--|:--|
| R1 | 高危词 | 关键词匹配 | 包含 KB-6 高危词表中任意词 |
| R2 | 词数硬限 | 计数 | 总英文词数 > 200 |
| R3 | 否定式指令 | 指令类型检查 | 句子为否定式指令（"don't show X""no water here"）→🔴 |
| | | | 描述性否定（"surface remains dry""zero scratches""untouched"）→✓ |
| | | | 数据性否定（"zero marks""no visible damage"）→✓ |
| | | | 状态性否定（"empty room""silent pause"）→✓ |
| R4 | 分镜编号 | 模式匹配 | 包含"镜头1/SHOT 1/分镜"等 |
| R5 | 时间码 | 模式匹配 | 包含"0:00-0:03/第1秒/00:01"等 |
| R6 | 复合运镜 | 模式匹配 | 同一句包含 ≥2 个运镜词 |
| R7 | 抽象材质 | 关键词匹配 | 材质位置使用"高端/高品质/漂亮/好用" |
| R8 | 主谓宾残缺 | 结构检查 | 句子无可渲染的主语或动作 |
| R9 | 单独情绪音 | 关键词匹配 | BGM/配乐/弦乐/钢琴曲作为独立指令 |

### 6.2 🟡 风险项（概率性，一致性 70-85%）

| 编号 | 检查项 | 触发条件 | Seedance 预测行为 |
|:--|:--|:--|:--|
| Y1 | 精确数量 | "3次""5下"等 | 数量不准 |
| Y2 | 精确时长 | "停顿2秒""持续3秒" | 时长不准 |
| Y3 | 文字生成 | 要求画面中生成指定文字 | 文字扭曲 |
| Y4 | 双人构图 | "两人对视""面对面" | 对视不准 |
| Y5 | 光线空间关系 | "光落在""light falls on" | 光线方向偏差 |
| Y6 | 状态变化 | 暗→亮/静→动 | 渐变过慢 |
| Y7 | 微表情 | "flinch""exhale" | 幅度偏差 |
| Y8 | 局部变形 | "纸板凹陷""表面弯曲" | 不执行 |
| Y9 | 词数区间 | 总词数 161-200 | 后段注意力衰减 |

### 6.3 编译器执行规则

```
步骤 1: 以句号/破折号为界切分句子
步骤 2: 逐句检查 R1-R9（🔴）
步骤 3: 逐句检查 Y1-Y9（🟡）
步骤 4: 汇总输出

判定:
  🔴 ≥ 2 → 触发修复（状态 4，仅替换触发句及依赖句）
  🔴 = 1 → 靶向修复该句 + 连贯性探针
  🔴 = 0 → 执行机械自检钩子 → 进入门控（状态 5）

修复优先级: R8(主谓宾) > R1(高危词) > R6(复合运镜) > R7(抽象材质) > R3(否定指令)
禁止: 改变 Hook 类型 / 删除角色行为 / 压缩段落
```

### 6.4 连贯性探针（修复后强制执行）

```
替换句后，执行以下 3 项检查:

□ 替换句的前一句是否自然引出替换句？（主语/动作/场景延续）
□ 替换句是否自然引出后一句？（主语/动作/场景延续）
□ 替换句的运镜词是否与前后句一致或合理过渡？

→ 全部 ✓ → 通过，回到状态 3
→ 任一 ✗ → 回滚替换，扩展修复范围至相邻句（≤3 句）
→ 扩展修复后仍 ✗ → 标记 ⚠️ 叙事连贯性风险，强制输出
```

### 6.5 机械自检钩子（编译器 🔴=0 后执行）

```
钩子 1: 词频扫描
  → 叙事中出现 ≥3 次的非功能词（排除 a/the/and/is/it/she/he）
  → 发现 → ⚠️ 标记（可能重复/啰嗦），附具体词及出现次数

钩子 2: 首尾物体呼应
  → 首句的核心物体（主语或关键道具）是否在末句/彩蛋句以物理形态再现？
  → 未再现 → ⚠️ 标记（叙事可能首尾断裂）
  → 例: 首句"纸箱"→末句"密封纸箱"=✓ / 首句"纸箱"→末句"火焰"=✗

钩子 3: 铁律逐条比对
  → 将叙事逐句与铁律 1-10 做机械比对
  → 每条铁律标注: 命中✓ / 未命中✗ / 不适用—
  → 输出: 铁律合规表（10 行）

→ 钩子输出仅附于快照末尾，不阻断流程
```

## 七、门控系统

### 7.1 技术门控（确定性，一致性 >95%）

```
□ 编译器 🔴 = 0
□ R3 未触发（无否定式指令）
□ R1 未触发（无高危词）
→ 全部通过 → 进入商业门控
```

### 7.2 商业门控（D1/D2/D7 为 3 级，一致性 70-85%）

```
□ D1 ≥ 🟡（Hook，门控特权）
□ D2 ≥ 🟡（产品价值）
□ D7 ≥ 🟡（CTA 转化）
→ 全部通过 → 进入辅助检查
→ 任一失败 → 靶向补丁（仅替换 CTA/Hook 相关句，≤1 句）→ 重新编译
```

### 7.3 D1/D2/D7 三级判定标尺

**D1 Hook 冲击力 — [M] 置信，门控特权**

| 等级 | 文本可观察信号（至少命中 2 条） |
|:--|:--|
| 🔴 | ① 前两句无声音词 ② 前两句为纯静态描述 ③ 前两句无动作动词 |
| 🟡 | ① 前两句包含声音事件词 ② 包含身体动作或物体运动 ③ 声音和动作在不同句中 |
| 🟢 | ① 声音和视觉在同句中同步 ② 存在亮度/音量突变 ③ 删除文字后画面仍独立传递 |

**D2 产品价值传达 — [H] 置信**

| 等级 | 文本可观察信号 |
|:--|:--|
| 🔴 | ① 产品/品牌未出现 ② 首次出现在最后 1/3 ③ 无法说出产品是什么 |
| 🟡 | ① 产品在前 1/3 出现 ② 关联 ≥1 可验证细节 ③ 品牌名通过道具可见 |
| 🟢 | ① 前 1/3 出现 + ≥3 可验证细节 ② 细节覆盖 ≥2 类别 ③ 竞品难复制 |

**D7 CTA 转化力 — [H] 置信**

| 等级 | 文本可观察信号 |
|:--|:--|
| 🔴 | ① 无品牌名 ② 无黑屏/标题画面 ③ 无法识别购买路径 |
| 🟡 | ① 品牌名至少出现 1 次 ② 有黑屏或标题画面 ③ 转化可通过评论区/Bio 实现 |
| 🟢 | ① 品牌通过道具自然出现 ② 黑屏含品牌名+标题或价格 ③ CTA 匹配客单价策略 |

**植入级别覆盖**：L4-L5 时 D7 允许 🟡（转化后置）。

### 7.4 映射置信度标注

门控通过后，输出 D1/D2/D7 的**文本→视频映射置信度**：

```
D1:🟢 → 视频实际 Hook 强度置信: 70%（取决于 Seedance 声音+视觉同步渲染）
D2:🟡 → 视频实际产品传达置信: 85%（文本可判定，视频层风险低）
D7:🟡 → 视频实际 CTA 可见性置信: 80%（取决于黑屏文字渲染质量）

校准依据: KB-14 Seedance 执行模式库
⚠️ 置信度基于 Seedance 2.0 已知行为模式，模型更新后需重新校准。
```

## 十、"还能更好吗"协议

```
触发: 用户说"还能更好吗"或同类意图

步骤 1: 扫描 D1/D2/D7 的当前等级，判断修复收益（高/中/低）
步骤 2: 存在高收益🟡 → 靶向补丁（≤1 句）→ 编译器快检
  → 补丁后执行变化检测:
    仅对比叙事段落（忽略技术参数/门控快照）
    IF 差异 < 5 个词 → 判定"无实质变化"
    → 直接输出 "⛔ 补丁无实质变化，已达框架最优。"
    IF 差异 ≥ 5 个词 → 输出 V_next
步骤 3: 全为中低收益或连续 2 轮无🟡→🟢转化 → 执行创意快检
步骤 4: 创意快检 4/4✓ → "⛔ 已达当前框架最优。继续优化属创意选择，非质量缺陷。"
步骤 5: 硬上限: 同一 Prompt 最多 3 次循环。第 4 次强制终止。
```

## 十二、动态 KB 路由表

### 12.1 路由规则

```
输入: 品类 + 目标市场
输出: 需加载的 KB 子集

始终加载（核心层，不依赖输入）:
  KB-1  Hook 模板
  KB-3  词汇库
```

## 十六、生产说明

### PS-1 Seedance 编译器执行协议

[同 v1.0，追加连贯性探针和自检钩子]

**完整审计流程**（仅 🔴≥1 时触发）：

```
步骤 1: 句子切分（以句号/破折号为界，标注每句词数）
步骤 2: 逐句扫描 🔴 项（R1-R9，R3 按指令类型判定）
步骤 3: 逐句扫描 🟡 项（Y1-Y9）
步骤 4: 汇总（🔴 计数 + 🟡 计数 + 逐句风险标注）
步骤 5: 判定
  🔴≥2 → 修复（≤2句）+ 连贯性探针
  🔴=1 → 靶向修复 + 连贯性探针
  🔴=0 → 机械自检钩子 → 门控
```

**编译器快径**：默认仅扫 🔴 → 🔴=0 直接输出🟡预案 + 自检钩子 → 门控

**修复规则**（仅替换触发句及依赖句，≤2 句）：

```
替换优先级: R8(主谓宾) > R1(高危词) > R6(复合运镜) > R7(抽象材质) > R3(否定指令)

替换方法:
  1. 不可见动作 → 可见动作（"exhales" → "steadies herself"）
  2. 抽象材质 → 具体材质（"高端金属" → "brushed stainless steel"）
  3. 空间光线 → 物体发光（"光落在她脸上" → "灯笼照亮了她的脸"）
  4. 微表情 → 大幅度动作（"flinches" → "stumbles back"）
  5. 精确控制 → 模糊范围（"3 knocks" → "knocks from the door"）
  6. 指定文字 → 视觉标记（"huopan.com" → "a distinct logo stamp"）

禁止: 改变情感弧线 / 删除角色行为 / 压缩段落 / 更换 Hook 类型
```
